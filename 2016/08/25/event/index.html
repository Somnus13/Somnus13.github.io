事件委托(代理)与事件传递

要了解事件委托和事件的传递，首先要了解

什么是事件

事件就是用户或者浏览器自身执行的某种动作，而响应某个事件的函就叫做事件处理程序（或事件侦听器）。
JavaScript赋予了我们动态创建页面的能力，而事件就是可以被JavaScript侦测到的行为。
网页中的每个元素都可以产生某些触发JavaScript函数的事件。
事件通常配合函数使用，所以我们就可以通过事件触发函数来操作页面。

介绍事件委托之前，先要介绍下JavaScript的事件传递机制。

4.通过事件监听函数绑定

addEventListener() 和 removeEventListener()
语法：elementObject.addEventListener(eventName,handle,usecapture);

参数 说明
elementObject 事件监听对象
eventName 事件名称 如 click，mouseover
handle 事件处理函数(函数名，无小括号)
useCapture 事件捕获还是冒泡，默认是false 冒泡。布尔类型
兼容 Chrome FF IE9+ √ IE6-8 ×

attachEvent() 和 detachEvent()
语法：elementObject.attachEvent(eventName,handle);

参数 说明
elementObject 事件监听对象
eventName 事件名称 如 onclick，onmouseover(注意区别 on)
handle 事件处理函数
兼容 Chrome FF ×

虽然两者均可，但是前者是W3C标准的事件监听函数

事件的传递与冒泡处理

DOM事件定义了两种事件流：捕获和冒泡。在成为标准之前，网景(Netscape)和微软选择了不同的方式实现了它们，
网景公司选择了捕获事件流，而微软则实现了冒泡事件流。到后来W3C标准实现的时候选择了组合两者。
默认情况下，事件使用冒泡流，而非捕获流。但是在一些浏览器里你可以显示指定使用捕获事件流，修改useCapture参数为true即可。

  • 冒泡事件流
    当事件在某一DOM元素被触发时，事件将随着节点继承自的各个父节点冒泡穿过整个DOM节点，至根节点。说白了就是子节点向根节点冒泡的过程。
    当我们实际代码工作中，我们有时需要来组织冒泡的传播。
    阻止冒泡的几种方法：
    1. return flase
    2. 调用event.target进行判断
    3. event.stopPropagation() -IE下 需要设置事件的cancelBubble属性为false;
  • 捕获事件流
    事件的处理将从DOM层次的根开始，而不是从触发事件的目标元素开始，事件被从目标元素的所有祖先元素依次往下传递。在这个过程中，事件会被从文档 根到事件目标元素之间各个继承派生的元素所捕获，如果事件监听器在被注册时设置了useCapture属性为true,那么它们可以被分派给这期间的任何 元素以对事件做出处理；否则，事件会被接着传递给派生元素路径上的下一元素，直至目标元素。事件到达目标元素后，它会接着通过DOM节点再进行冒泡。

注解：attachEvent()事件处理函数内部中 this指向的是window对象，且同一事件处理函数只能绑定一次。

事件委托与代理

事件委托，通俗的将就是把当前元素需要处理的事件委托给别人来做，当然在html中这个别人不是随便的别人，而是当前元素的父级元素。原理呢就是利用之前所说的冒泡，来把事件添加到父级元素来触发执行。
优点：提高性能(简短的四个字里多少血泪QAQ)
举个栗子，经常和页面打交道，难免要操作li。例如什么划过变色啊，什么点击弹出相应索引值啊等等。一般呢我们采用的方法就是遍历li，然后每个添加事件，问题是li个数较少这没啥问题，如果数量级增加一位呢？
所以要通过事件委托来提高性能。
