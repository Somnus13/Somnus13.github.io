移动端fixed定位的坑

IOS 当fixed 遇到 input

背景

  • 触发前提：ios系统 fixed定位 input或者textarea元素
  • 触发：页面input或textarea(以下统称为inputs) foucus触发激活软键盘
  • 结果：fixed定位元素位置偏移

深坑重现

移动端页面难免会有些固定底部或顶部的内容块，当这些页面遇到inputs，问题就来了。
代码如下：

inputs触发focus时，UC效果如下：


不仅位置发生了偏移，还遮挡了输入框。

解决

然后求助谷歌百度等再结合前人经验(毙掉iscroll方案)，自己拿出了三套方案来解决，总的来说各有优劣。

方案一：整体内容position:absolute定位，高度100%

貌似这是大多数人的解决方案，但是由于公司内部项目模板问题，absolute方案需要大幅动工，so pass！

方案二：触发修改fixed为relative，失去焦点还原

实现效果：焦点触发，fixed元素回归文档流，自动置于下方，失去焦点还原

方案三：fixed定位，js计算fixed元素top值并调整位置

借鉴自前人代码，加以修改。

//注：软键盘呼出是会触发resize事件的
var isBottom = true;

var topReset = function() {

    //解绑resize事件  以免进入死循环
    $(window).unbind("resize", topReset);

    var t = document.documentElement.scrollTop||document.body.scrollTop, fixed_height = 44, top;

    //网页可见高度加上滚动条高度  - fixed元素的高度
    top = window.innerHeight+ t - fixed_height;

    //设置fixed div的top
    $(".footer").css({"top": top+"px" });

    //重新绑定resize事件
    setTimeout(function() {
        $(window).bind("resize", topReset);
    }, 5);
}


if (isBottom) {
    topReset();
    function touchstart(event) {
        $("footer").css("opacity",0);
    }
    function touchend(event) {
        $("footer").animate({"opacity":1});
    }
    //touch的时候隐藏
    document.addEventListener("touchstart", touchstart, false);
    //滚动后重新设置fixed div的位置
    window.onscroll = function(){
        var scrollTop = $(this).scrollTop();
　　    var scrollHeight = $(document).height();
　　    var windowHeight = $(this).height();
    　　if(scrollTop + windowHeight == scrollHeight){
    　　　　isIOS = false;
    　　}else{
            isIOS = true;
        }
        topReset()
    };
    //touch后显示
    document.addEventListener("touchend", touchend, false);
}

//所有input blur时重新计算位置

$("section").bind("focusout", "input", topReset);

最后一种没采用的原因就是：js计算不够灵敏，而且一直监听页面inputs，计算top值，效率不高，而且有个小bug(第一次触发，位置还是不对)，建议不采用。

综上，对于移动前端来讲，这是难免会遇到的坑，花了大半天时间的成果还是记下笔记比较好。
